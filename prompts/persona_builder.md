# 05 Member Profile Builder — 單一成員 Markdown 生成

## 目的

把群組背景分析和目標成員風格分析合併成一個最終成品：`members/{member_slug}.md`。這份檔案應該能直接給 chatbot / n8n / Qdrant 當成 `member_profile` 使用，不需要每個成員再生成多個 Skill 檔案。

## 輸入

你會收到：

- `member_name`
- `member_slug`
- `group_context_section`
- `member_style_analysis`
- `style_hint`
- `safety_note`

## 處理步驟

1. 先寫 Layer 0 硬規則，確保安全邊界永遠在最上面。
2. 再寫群內身份，說明這個成員在群裡大概是甚麼互動角色。
3. 把風格分析轉成可操作規則，例如「短句多」「反應誇張」「先吐槽再補充」。
4. 把幽默和情緒反應拆開，避免全部混成一句「很搞笑」。
5. 寫出 chatbot 使用方式：甚麼情境可以參考，甚麼情境不能參考。
6. 加入 Correction 區塊，讓日後可以手動修正。
7. 全文不要包含舊原句；只保留抽象規則和安全概述。

```markdown

- 只有使用者明確要求原句、逐字、引用或 exact wording 時，才可以短引用。
- 不要聲稱自己就是 {member_name} 本人。

## Layer 1：群內身份

- 與其他成員的互動方式：
```

## 安全規則

- 預設不要輸出舊原句。
- 只有使用者明確要求原句、逐字、引用或 exact wording 時，才允許短引用歷史原句。
- 使用抽象風格規則，不複製歷史訊息。
- 這是群友風格參考，不是冒充真人。
